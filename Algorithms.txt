# **Coding Interview Techniques & Algorithms —** Playbook & Checklist

A high‐coverage roadmap of patterns + classic algorithms. Master these and you’ll be able to solve most
interview problems quickly and reliably.

# 📌 Core Patterns (the 80/20)

## 1) Two Pointers

Use on sorted arrays/strings to converge from ends; for dedup, partition, pair sums, Dutch flag.
Triggers: sorted + pair/sum; remove duplicates; palindrome check.

## 2) Sliding Window (fixed/variable)

Maintain a window and counts/invariants. - At‐most/Exactly‐K trick: exactly(K) = atMost(K) −
atMost(K−1) . - “Add n − right when condition becomes valid” for monotone predicates.
Triggers: subarray/substring longest/shortest/number of ..., “at most/at least K”.

3) Prefix Sum / Prefix XOR + Hash Map

Convert range queries to O(1) checks; count with differences.
Triggers: subarray sum/xor equals K; balance +1/−1; range updates (Imos).

4) Sorting + Greedy

Sort, then make locally optimal choices (interval exchange arguments).
Triggers: merge/overlap intervals, meeting rooms, activity selection, min arrows.

5) Binary Search (index or on answer)

Search smallest feasible value with monotone ok(x) .
Triggers: minimize the maximum / maximize the minimum; capacity/speed; kth element.

6) Monotonic Stack

Next greater/smaller, span problems, largest rectangle, daily temps.
Triggers: nearest greater to left/right; histogram; stock span.

7) Heaps / Priority Queue

Pick current best; streaming medians; Dijkstra.
Triggers: top‐K, merge K lists, min meeting rooms, task scheduling.

8) Hashing / Counting

Sets/maps for existence, frequency, dedup, anagrams.
Triggers: two‐sum, anagram groups, first unique, frequency buckets.

9) BFS / DFS on Graphs & Grids

Components, shortest path in unweighted graphs, flood fill.
Triggers: number of islands, word ladder (BFS), cycle detection (DFS).

10) Union‐Find (Disjoint Set Union)

Connectivity, cycle detection, Kruskal MST.
Triggers: dynamic connectivity, redundant connection, accounts merge.

11) Trees (Basics + LCA)

Traversals, subtree sums, diameters; BST inorder = sorted.
Triggers: kth in BST, path sums, distance queries; LCA (binary lifting).

12) Dynamic Programming (fundamentals)

Memo/tabulation for overlapping subproblems & optimal substructure.
Families: knapsack, coin change, LIS (patience sorting), LCS/edit distance, grid paths.
Triggers: “min/max cost/ways”; choose/split; sequence alignments.

13) Backtracking / Search with Pruning

Generate permutations/combos; constraint search.
Triggers: N‐Queens, combination sum, word search, phone keypad.

14) Math & Bit Tricks

XOR, bitmasks, popcount, power‐of‐two, GCD/LCM, fast pow, sieve.
Triggers: subset enumeration, parity, toggling; number theory checks.

15) String Toolbelt (core)

KMP/Z for pattern search; rolling hash (Rabin–Karp) for duplicates; Trie for prefixes.
Triggers: fast substring search, repeated substrings, autocomplete.

🎯 Intermediate / High‐ROI Extras

16) Monotonic Queue (Deque) — sliding window min/max; DP speedups with convex/monotone structure.
17) Topological Sort + DP on DAGs — scheduling, longest path on DAG.
18) Range DS — Fenwick (BIT), Segment Tree (+ lazy), Sparse Table (idempotent).
19) Shortest‐Path Variants — Dijkstra, 0‐1 BFS, Bellman‐Ford, Floyd‐Warshall.

20) Flow & Matching — Bipartite matching (Hopcroft–Karp), Max‐Flow (Dinic/Edmonds‐Karp), Min‐cut.

22) Advanced DP Optimizations — Divide‐and‐Conquer DP, Knuth, Convex Hull Trick.
23) Meet‐in‐the‐Middle — split set, combine (e.g., subset sums near 2^20 scale).

25) Geometry & Sweep Line — orientation, interval/segment sweep, event sorting.

🧭 Quick Mapping: Clues → Techniques

- “subarray/substring at most/exactly K” → Sliding window; or Prefix+Hash for sums/XOR.
• “count subarrays sum==k / xor==k” → Prefix + HashMap.
• “min rooms / overlap / merge” → Sort + Greedy / Heap.
• “minimize max / maximize min” → Binary search on answer.
- “next greater / histogram” → Monotonic stack.
• “unweighted shortest path” → BFS; weighted → Dijkstra; {0,1} weights → 0‐1 BFS.
• “kth smallest / running median” → Heaps / Quickselect.
• “connectivity / cycles” → DSU or DFS.
• “dependencies” → Toposort (+ DP on DAG).
• “range query + updates” → Fenwick/SegTree (lazy); idempotent RMQ → Sparse Table.
• “generate all with constraints” → Backtracking + pruning.
• “longest palindrome” → Manacher; “duplicate substrings” → Rolling hash / Suffix array.

📚 Notable Algorithms to Prepare (with one‐liners)
Graphs & Trees

- Kruskal’s MST — sort edges + DSU; O(E log E).
• Prim’s MST — grow from a node using a min‐heap; O(E log V).
• Dijkstra — non‐negative weights shortest paths; min‐heap; O(E log V).
• 0‐1 BFS — weights {0,1}; deque; O(V+E).
• Bellman‐Ford — handles negative edges; O(VE).
• Floyd‐Warshall — all‐pairs shortest paths; O(V^3) (small V).
• Topological Sort — Kahn’s or DFS order for DAGs.
• Union‐Find (DSU) — union by rank + path compression; near O(α(N)).
• Tarjan/Kosaraju — strongly connected components in directed graphs.
• LCA (Binary Lifting / Euler + RMQ) — ancestor queries in O(log N)/O(1).

Strings

- Horspool (Boyer–Moore–Horspool) — simplified BM with bad‐char shift; fast in practice.
• Boyer–Moore (full) — bad‐character + good‐suffix heuristics.
• KMP — linear time pattern matching via prefix function (lps).
• Rabin–Karp — rolling hash for multiple/pattern search; duplicates.
- Z‐Algorithm — compute Z‐array; alternative to KMP for matching.
• Aho–Corasick — multi‐pattern automaton with failure links.
• Manacher — longest palindromic substring in O(n).
• Suffix Array + LCP (Kasai) / Suffix Automaton — distinct substrings, repeats.

Range / Arrays / Math

- Fenwick (BIT) — point update + prefix sum in O(log N).
• Segment Tree (+ Lazy) — range query & update in O(log N).
• Sparse Table — static idempotent queries (min/gcd) in O(1).

- Sweep Line — event‐based interval/geometry counting.
• Quickselect — average O(n) kth‐element; partition‐based.
• Patience Sorting (LIS O(n log n)) — LIS length with predecessors to rebuild.
• Euclid’s GCD / Extended GCD — gcd, modular inverses (with prime mod via Fermat).
• Fast Exponentiation — pow in O(log n); modular variants.
• Sieve of Eratosthenes — primes up to N in O(N log log N).
• Binary Search on Answer — parametric search framework.

🧩 Reusable Mini‐Templates (pseudocode)
Sliding Window (at least K with monotone predicate)

```
  ans = 0; left = 0
  for right in [0..n):

```

```
      add nums[right] to window state
      while predicate(window) is true:

```

```
          ans += n - right   # all longer tails also valid
          remove nums[left] from window state
          left += 1

```

return ans

Prefix Sum + HashMap (sum == K)

```
  count = {0:1}; pref = 0; ans = 0
  for x in nums:

```

```
      pref += x
      ans += count.get(pref - K, 0)
      count[pref] = count.get(pref, 0) + 1

```

return ans

🧠 Complexity & Constraints: Quick Heuristics

- n ≈ 1e5 → aim for O(n) or O(n log n); avoid O(n2).
• Use monotonicity to justify binary search on answer or the + (n − right) window trick.
• Prefer counting/hash maps over value‐indexed arrays when values are large (1e9).
• Pre‐sort when greedy needs order; avoid re‐sorting in loops.
