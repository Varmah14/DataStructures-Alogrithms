🚀 Monotonic Stack — Complete Guide (Concept → How to Spot → How to Implement → Problems)
1) What is it?

A stack that maintains a monotone order (increasing or decreasing) while scanning a sequence. It lets you solve “next/previous greater/smaller” and “first boundary” problems in O(n) by ensuring each element is pushed/popped at most once.

2) When to use (fast recognition)

Trigger phrases / clues

“Next/previous greater/smaller element (to left/right)”

“Nearest greater/smaller”

“Histogram, largest rectangle”

“Daily temperatures, stock span”

“Sum of subarray minimums/maximums”

“Find first boundary that violates X”

If you need the first index to the left/right that is ≥, >, ≤, < the current value → it’s almost certainly a monotonic stack.

3) Mental model

Monotone Increasing stack (values up): pop while top > cur.
Finds next smaller (right boundary where value drops).

Monotone Decreasing stack (values down): pop while top < cur.
Finds next greater (right boundary where value rises).

Store indices, not values — you often need distances/widths.

4) Core templates (copy-paste)
A) Next Greater to the Right (NGE)
def next_greater_right(nums):
    n = len(nums)
    res = [-1]*n
    st = []  # indices; keep values in decreasing order
    for i, x in enumerate(nums):
        while st and nums[st[-1]] < x:
            j = st.pop()
            res[j] = x            # or set to i for index
        st.append(i)
    return res

B) Next Smaller to the Right (NSR)

C) Largest Rectangle in Histogram (LC 84)
def largestRectangleArea(h):
    h.append(0)             # sentinel to flush
    st, best = [-1], 0      # stack of indices; -1 as left sentinel
    for i, x in enumerate(h):
        while st[-1] != -1 and h[st[-1]] > x:
            height = h[st.pop()]
            width  = i - st[-1] - 1
            best = max(best, height * width)
        st.append(i)
    return best

D) Daily Temperatures (LC 739)
def dailyTemperatures(T):
    n = len(T)
    res = [0]*n
    st = []  # decreasing temps (indices)
    for i, t in enumerate(T):
        while st and T[st[-1]] < t:
            j = st.pop()
            res[j] = i - j
        st.append(i)
    return res

E) Sum of Subarray Minimums (LC 907) — tie-handling!
def sumSubarrayMins(arr):
    MOD = 10**9 + 7
    n = len(arr)

    # prev strictly smaller: use '>' to keep last strictly smaller to LEFT
    prev = [-1]*n
    st = []
    for i, x in enumerate(arr):
        while st and arr[st[-1]] > x:
            st.pop()
        prev[i] = st[-1] if st else -1
        st.append(i)

    # next smaller-or-equal: use '>=' on RIGHT to break ties on the right
    nxt = [n]*n
    st = []
    for i, x in enumerate(arr):
        while st and arr[st[-1]] >= x:
            j = st.pop()
            nxt[j] = i
        st.append(i)

    ans = 0
    for i, x in enumerate(arr):
        L = i - prev[i]   # choices on the left
        R = nxt[i] - i    # choices on the right
        ans = (ans + x * L * R) % MOD
    return ans


Rule of thumb for ties:
For “sum of minimums”: use > on left and >= on right (or the opposite pairing) to avoid double-counting equals. For “sum of maximums,” flip the inequalities.

5) Pitfalls checklist

Use indices, not raw values (you need distances/widths).

Pick strict vs non-strict comparisons carefully (tie strategy).

Remember sentinels (e.g., 0 at end for LC 84) to flush the stack.

Compute answers on pop (that’s when both boundaries are known).

For circular arrays: iterate twice or use i % n.
